Getting Started

Quick Start

1

Clone the repo

git clone git@github.com:barakbl/nulama.git
cd nulama
2

Generate a TOML spec from any CLI tool

# Inside Claude Code (from this directory)
/nu_create_toml_from_posix_command docker
3

Generate Nushell commands from the spec

/to_nu docker.toml
4

Load and use in Nushell

# Source in current session
source docker.nu

# Filter running containers — structured, not text
nudocker ps --all | where status =~ "Up"
nugit log --max-count 10 | where author == "Alice"

No clone needed — fetch directly from GitHub

http get https://raw.githubusercontent.com/barakbl/nulama/main/examples/brew/brew.nu | save brew.nu
source brew.nu
Claude Code Slash Commands

Commands

/nu_create_toml_from_posix_command <tool>

Inspects a POSIX CLI tool and generates a TOML spec file.

Runs <tool> --help, asks which subcommands and flags to include, inspects each selected subcommand, and writes <tool>.toml to the current directory.

/to_nu <path-to-toml>

Reads a TOML spec and generates Nushell def blocks with structured output parsing, tab completion, and streaming support.

If the TOML doesn't exist yet, /to_nu will offer to create it automatically.

What You Get

Features

⌨️

Tab Completion

Native Nushell def blocks give automatic tab completion for all subcommands and flags. No scripts to install.

📊

Structured Output

Tabular detection, key-value parsing, JSON, and line-oriented strategies turn raw text into real records.

🌊

Streaming Mode

Follow flags stream raw output; non-follow calls parse into tables. Works with docker logs -f and more.

🔒

Null-safe Capture

Safe complete access with default fallbacks and empty-output guards prevents runtime errors.

🏷️

Header Sanitization

CONTAINER ID becomes container_id automatically — ready for Nushell pipelines.

📋

Structured Help

Every wrapper exposes --info (-i) that returns the tool's help as a filterable table.

Usage examples

# Structured help — filterable table
nubrew -i | where section == "Troubleshooting"

# Docker — filter running containers
nudocker ps --all | where status =~ "Up"

# Docker — sort images by size
nudocker images | sort-by size

# Docker — search error logs
nudocker logs --tail 50 my-container | where line =~ "ERROR"

# Git — structured log
nugit log --max-count 10 | where author == "Alice"

# pip — outdated packages
nupip list --outdated | sort-by version
